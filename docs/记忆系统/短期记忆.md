# 短期记忆 (ShortTermMemory)

**源文件**: `context/short_term_memory.py`

## 概述

短期记忆是一个纯内存的滑动窗口，存储当前会话最近 N 轮对话（默认 10 轮），用于为 Agent 提供上下文理解和消歧能力。进程结束后数据丢失，不做持久化。

## 数据结构

每条消息以 `Dict[str, Any]` 存储，结构如下：

```python
{
    "role": str,        # "user" 或 "assistant"
    "content": str,     # 消息文本
    "timestamp": str,   # ISO 8601 格式时间戳，如 "2026-04-04T10:30:00.123456"
    "metadata": dict    # 自定义元数据，默认为空字典
}
```

内部用一个 `List[Dict[str, Any]]` (`self.messages`) 按时间顺序存放所有消息。

---

## 类: `ShortTermMemory`

### `__init__(self, max_turns: int = 10)`

初始化短期记忆实例。

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `max_turns` | `int` | `10` | 最大保存轮数。一轮 = 一对 user + assistant 消息 |

**内部状态**:
- `self.max_turns`: 保存传入的最大轮数
- `self.messages`: 初始化为空列表 `[]`

---

### `add_message(self, role: str, content: str, metadata: Dict = None)`

向短期记忆添加一条消息，并在超出容量时自动淘汰最旧的消息。

**输入**:

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `role` | `str` | (必填) | 消息角色，`"user"` 或 `"assistant"` |
| `content` | `str` | (必填) | 消息文本内容 |
| `metadata` | `Dict` | `None` | 可选的额外元数据，传 `None` 时存为 `{}` |

**输出**: 无返回值。

**逻辑**:
1. 构造消息字典，包含 `role`、`content`、当前时间戳（`datetime.now().isoformat()`）、`metadata`（若为 `None` 则替换为 `{}`）。
2. 将消息追加到 `self.messages` 列表末尾。
3. **滑动窗口淘汰**: 计算最大消息数 `max_messages = max_turns * 2`（每轮 2 条）。若当前消息数超过该上限，则对列表做尾部切片 `self.messages[-max_messages:]`，丢弃最早的消息。
4. 记录 debug 日志。

**示例**:
```python
mem = ShortTermMemory(max_turns=2)  # 最多保存 4 条消息
mem.add_message("user", "你好")
mem.add_message("assistant", "你好！有什么可以帮你的？")
mem.add_message("user", "调研一下AI市场")
mem.add_message("assistant", "好的，正在调研...")
mem.add_message("user", "结果如何？")  # 此时第1条("你好")被淘汰
```

---

### `get_recent_context(self, n_turns: int = None) -> List[Dict[str, Any]]`

获取最近 N 轮对话的消息列表。

**输入**:

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `n_turns` | `int` | `None` | 获取的轮数。`None` 表示返回全部消息 |

**输出**: `List[Dict[str, Any]]` — 消息字典列表的**副本**（不会修改内部状态）。

**逻辑**:
1. 若 `n_turns` 为 `None`，返回 `self.messages` 的浅拷贝。
2. 否则计算 `n_messages = n_turns * 2`。
3. 若当前消息数 > `n_messages`，返回尾部 `n_messages` 条；否则返回全部消息的浅拷贝。

---

### `get_context_string(self, n_turns: int = 5) -> str`

将最近对话格式化为人类可读的字符串，供 Agent prompt 拼接使用。

**输入**:

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `n_turns` | `int` | `5` | 获取的轮数 |

**输出**: `str` — 格式化的对话文本。若无消息则返回 `"无历史对话"`。

**逻辑**:
1. 调用 `self.get_recent_context(n_turns)` 获取消息列表。
2. 若列表为空，返回 `"无历史对话"`。
3. 遍历每条消息，将 `role` 映射为中文（`"user"` → `"用户"`，其他 → `"助手"`），拼接为 `"{角色}: {内容}"` 格式。
4. 用 `\n` 连接所有行并返回。

**输出示例**:
```
用户: 调研一下AI市场
助手: 好的，正在为您调研AI市场...
用户: 结果如何？
```

---

### `clear(self)`

清空所有短期记忆。

**输入**: 无参数。

**输出**: 无返回值。

**逻辑**: 将 `self.messages` 重置为空列表 `[]`，记录 info 日志。

---

### `get_statistics(self) -> Dict[str, Any]`

返回当前短期记忆的统计信息。

**输入**: 无参数。

**输出**: `Dict[str, Any]`，结构如下：

```python
{
    "total_messages": int,       # 当前存储的消息总数
    "max_turns": int,            # 配置的最大轮数
    "oldest_message_time": str | None,  # 最早消息的时间戳，无消息时为 None
    "newest_message_time": str | None   # 最新消息的时间戳，无消息时为 None
}
```

## 使用方式

### 基本用法

```python
from context.short_term_memory import ShortTermMemory

memory = ShortTermMemory(max_turns=10)

# 添加对话
memory.add_message("user", "帮我调研一下竞品")
memory.add_message("assistant", "好的，正在为您调研竞品信息...")

# 获取最近 3 轮上下文
recent = memory.get_recent_context(n_turns=3)

# 获取格式化字符串（用于拼接到 Agent prompt）
context_str = memory.get_context_string(n_turns=5)

# 查看统计
stats = memory.get_statistics()
```

### 在 MemoryManager 中的集成

短期记忆通常不直接使用，而是通过 `MemoryManager` 统一管理：

```python
from context.memory_manager import MemoryManager

mm = MemoryManager(user_id="user_001")
mm.add_message("user", "你好")  # 同时写入短期记忆
context = mm.get_context()       # 合并短期 + 长期记忆
```

## 设计要点

- **滑动窗口**: 仅保留最近 `max_turns` 轮（默认 10 轮 = 20 条消息），超出自动丢弃最早的消息。
- **纯内存**: 不做任何持久化，进程结束即丢失。持久化由长期记忆负责。
- **返回副本**: `get_recent_context` 返回列表的浅拷贝，外部修改不影响内部状态。
- **角色映射**: `get_context_string` 中非 `"user"` 的角色统一映射为 `"助手"`。
