# 长期记忆 (LongTermMemory)

**源文件**: `context/long_term_memory.py`

## 概述

长期记忆是基于 JSON 文件的持久化存储层，按用户维度保存项目偏好、聊天历史和项目决策记录，支持跨会话访问。每个用户对应一个独立的 JSON 文件，路径为 `data/memory/{user_id}.json`。

## 持久化文件结构

每个用户的 JSON 文件完整结构如下：

```json
{
  "user_id": "user_001",
  "created_at": "2026-04-04T10:00:00.000000",
  "updated_at": "2026-04-04T12:30:00.000000",
  "preferences": [
    {"type": "target_market", "value": "B2B SaaS"},
    {"type": "tech_stack", "value": "Python + React"}
  ],
  "chat_history": [
    {
      "role": "user",
      "content": "调研一下AI市场",
      "timestamp": "2026-04-04T10:05:00.000000",
      "session_id": "session_abc123"
    }
  ],
  "project_history": [
    {
      "record_id": "record_1",
      "timestamp": "2026-04-04T10:10:00.000000",
      "action_type": "research",
      "description": "完成AI市场调研"
    }
  ],
  "statistics": {
    "total_projects": 1,
    "total_messages": 2,
    "frequent_actions": {"research": 3, "marketing": 1}
  }
}
```

---

## 类: `LongTermMemory`

### `__init__(self, user_id: str, storage_path: str = "data/memory")`

初始化长期记忆，加载或创建用户的 JSON 文件。

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `user_id` | `str` | (必填) | 用户 ID，决定文件名 |
| `storage_path` | `str` | `"data/memory"` | 存储目录路径 |

**内部状态**:
- `self.user_id`: 用户 ID
- `self.storage_path`: 存储目录
- `self.db_path`: 拼接为 `{storage_path}/{user_id}.json`
- `self.data`: 加载后的完整数据字典

**逻辑**:
1. 用 `Path.mkdir(parents=True, exist_ok=True)` 确保存储目录存在。
2. 调用 `self._load()` 加载数据：若文件存在则读取并做数据迁移；若不存在则初始化空结构。

---

### `_load(self) -> Dict[str, Any]`

从 JSON 文件加载数据。

**输入**: 无参数。

**输出**: `Dict[str, Any]` — 用户的完整数据字典。

**逻辑**:
1. 若 `self.db_path` 文件存在，读取 JSON 并调用 `_ensure_fields()` 补全可能缺失的字段，返回补全后的数据。
2. 若读取失败（JSON 损坏等），捕获异常，返回 `_init_data()` 的空结构。
3. 若文件不存在，返回 `_init_data()` 的空结构。

---

### `_ensure_fields(self, data: Dict[str, Any]) -> Dict[str, Any]`

确保必需字段存在，防止手动编辑 JSON 导致字段缺失。

**输入**:

| 参数 | 类型 | 说明 |
|---|---|---|
| `data` | `Dict[str, Any]` | 从文件读取的原始数据 |

**输出**: `Dict[str, Any]` — 补全后的数据。

**逻辑**:
1. 调用 `_init_data()` 获取默认结构，遍历其所有顶层 key，若 `data` 中缺失则用默认值填充。
2. 额外检查 `statistics.total_messages` 是否存在，不存在则补 `0`。

---

### `_init_data(self) -> Dict[str, Any]`

创建空的初始数据结构。

**输入**: 无参数。

**输出**: `Dict[str, Any]` — 包含所有必需字段的空数据结构（见上方"持久化文件结构"）。

---

### `_save(self)`

将当前 `self.data` 写入 JSON 文件。

**输入**: 无参数。

**输出**: 无返回值。

**逻辑**:
1. 更新 `self.data["updated_at"]` 为当前时间。
2. 以 UTF-8 编码写入文件，`ensure_ascii=False` 保留中文，`indent=2` 格式化。
3. 写入失败时捕获异常并记录错误日志（不抛出）。

**注意**: 每次写操作都是全量覆盖写入，不是增量更新。

---

### `save_preference(self, pref_type: str, value: Any)`

保存或更新一条项目偏好。

**输入**:

| 参数 | 类型 | 说明 |
|---|---|---|
| `pref_type` | `str` | 偏好类型，如 `"target_market"`、`"tech_stack"`、`"brand_style"` |
| `value` | `Any` | 偏好值 |

**输出**: 无返回值。

**逻辑**:
1. 遍历 `self.data["preferences"]` 列表，查找 `type` 匹配的项。
2. 若找到，**原地更新** `value`。
3. 若未找到，追加新项 `{"type": pref_type, "value": value}`。
4. 调用 `_save()` 持久化。

**示例**:
```python
mem = LongTermMemory("user_001")
mem.save_preference("target_market", "B2B SaaS")     # 新增
mem.save_preference("target_market", "B2C 消费者")    # 更新已有项
```

---

### `get_preference(self, pref_type: str = None) -> Any`

获取项目偏好。

**输入**:

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `pref_type` | `str` | `None` | 偏好类型。`None` 返回全部偏好 |

**输出**:
- 当 `pref_type=None` 时：`Dict[str, Any]` — 所有偏好的字典形式 `{"target_market": "B2B", "tech_stack": "Python"}`
- 当指定 `pref_type` 时：对应的 `value`，未找到则返回 `None`

**逻辑**:
1. 若 `pref_type` 为 `None`，遍历偏好列表，构造 `{type: value}` 字典返回。
2. 否则遍历查找匹配项，返回其 `value`，找不到返回 `None`。

---

### `add_chat_message(self, role: str, content: str, session_id: str = None)`

添加一条聊天消息到长期记忆并持久化。

**输入**:

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `role` | `str` | (必填) | `"user"` 或 `"assistant"` |
| `content` | `str` | (必填) | 消息文本 |
| `session_id` | `str` | `None` | 所属会话 ID |

**输出**: 无返回值。

**逻辑**:
1. 构造消息字典：`{"role", "content", "timestamp", "session_id"}`。
2. 追加到 `self.data["chat_history"]`。
3. `statistics.total_messages` 自增 1。
4. 调用 `_save()` 持久化。

**注意**: 与短期记忆不同，长期记忆**没有滑动窗口**，消息无上限、不会自动淘汰。文件会随使用持续增长。

---

### `get_chat_history(self, limit: int = None, session_id: str = None) -> List[Dict[str, Any]]`

获取聊天历史。

**输入**:

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `limit` | `int` | `None` | 返回最近 N 条，`None` 返回全部 |
| `session_id` | `str` | `None` | 按会话过滤，`None` 返回所有会话 |

**输出**: `List[Dict[str, Any]]` — 消息列表。

**逻辑**:
1. 若指定 `session_id`，先过滤出该会话的消息。
2. 若指定 `limit`，返回尾部 `limit` 条；否则返回全部。

**注意**: 返回的是原始列表的切片，不是副本。外部修改会影响内部状态。

---

### `save_project_history(self, project_info: Dict[str, Any])`

保存一条项目决策记录。

**输入**:

| 参数 | 类型 | 说明 |
|---|---|---|
| `project_info` | `Dict[str, Any]` | 项目决策信息，通常包含 `action_type`、`description` 等字段 |

**输出**: 无返回值。

**逻辑**:
1. 构造记录：自动生成 `record_id`（`"record_{序号}"`）和 `timestamp`，与传入的 `project_info` 合并（`**` 展开）。
2. 追加到 `self.data["project_history"]`。
3. `statistics.total_projects` 自增 1。
4. 若 `project_info` 中有 `action_type`，更新 `statistics.frequent_actions` 的计数。
5. 调用 `_save()` 持久化。

**示例**:
```python
mem.save_project_history({
    "action_type": "research",
    "description": "完成AI教育市场调研",
    "result_summary": "市场规模约500亿"
})
# 存储结果:
# {"record_id": "record_1", "timestamp": "...", "action_type": "research", "description": "...", "result_summary": "..."}
```

---

### `get_project_history(self, limit: int = 10) -> List[Dict[str, Any]]`

获取项目决策历史。

**输入**:

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `limit` | `int` | `10` | 返回最近 N 条，`0` 或 `None` 返回全部 |

**输出**: `List[Dict[str, Any]]` — 项目决策记录列表。

**逻辑**: 若 `limit` 为真值，返回尾部 `limit` 条；否则返回全部。

---

### `get_frequent_actions(self, top_n: int = 5) -> List[tuple]`

获取最常用的操作类型排行。

**输入**:

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `top_n` | `int` | `5` | 返回前 N 个 |

**输出**: `List[tuple]` — `[(action_type, count), ...]`，按 count 降序排列。

**逻辑**:
1. 取出 `statistics.frequent_actions` 字典。
2. 按 value 降序排序。
3. 返回前 `top_n` 个元组。

**示例**:
```python
mem.get_frequent_actions(3)
# [("research", 5), ("marketing", 3), ("product", 1)]
```

---

### `get_statistics(self) -> Dict[str, Any]`

返回统计信息的副本。

**输入**: 无参数。

**输出**: `Dict[str, Any]` — `statistics` 字段的浅拷贝，结构如下：

```python
{
    "total_projects": int,
    "total_messages": int,
    "frequent_actions": {"research": 5, "marketing": 3, ...}
}
```

---

### `clear_history(self)`

清空聊天历史和项目历史，但**保留偏好设置**。

**输入**: 无参数。

**输出**: 无返回值。

**逻辑**:
1. 清空 `chat_history` 和 `project_history` 列表。
2. 重置 `statistics` 中的 `total_projects`、`total_messages` 为 0，`frequent_actions` 为空字典。
3. 调用 `_save()` 持久化。

---

### `delete_all(self)`

删除该用户的整个 JSON 文件。

**输入**: 无参数。

**输出**: 无返回值。

**逻辑**: 若文件存在，调用 `os.remove()` 删除。不清空内存中的 `self.data`。

**注意**: 调用后 `self.data` 仍然保留在内存中。若后续再调用 `_save()` 会重新创建文件。

---

## 使用方式

### 基本用法

```python
from context.long_term_memory import LongTermMemory

mem = LongTermMemory(user_id="user_001", storage_path="data/memory")

# 保存偏好
mem.save_preference("target_market", "B2B SaaS")
mem.save_preference("tech_stack", "Python + React")

# 读取偏好
all_prefs = mem.get_preference()          # {"target_market": "B2B SaaS", "tech_stack": "Python + React"}
market = mem.get_preference("target_market")  # "B2B SaaS"

# 添加聊天消息
mem.add_chat_message("user", "调研AI市场", session_id="session_001")
mem.add_chat_message("assistant", "调研结果...", session_id="session_001")

# 保存项目决策
mem.save_project_history({
    "action_type": "research",
    "description": "完成AI市场调研"
})

# 查询
history = mem.get_chat_history(limit=10)
projects = mem.get_project_history(limit=5)
top_actions = mem.get_frequent_actions(3)
stats = mem.get_statistics()

# 清理
mem.clear_history()   # 清历史，保留偏好
mem.delete_all()      # 删除整个文件
```

### 在 MemoryManager 中的集成

长期记忆通常不直接使用，而是通过 `MemoryManager` 统一管理：

```python
from context.memory_manager import MemoryManager

mm = MemoryManager(user_id="user_001", session_id="session_001")
mm.add_message("user", "你好")  # 同时写入短期记忆 + 长期记忆
```

`MemoryManager.add_message()` 内部同时调用 `short_term.add_message()` 和 `long_term.add_chat_message()`，短期记忆不带 `session_id`，长期记忆带 `session_id`。

## 设计要点

- **全量写入**: 每次修改操作（`save_preference`、`add_chat_message`、`save_project_history`、`clear_history`）都会触发 `_save()`，将整个 `self.data` 全量写入文件。高频写入场景下可能有性能问题。
- **无容量上限**: `chat_history` 和 `project_history` 没有滑动窗口，会无限增长。长期使用后 JSON 文件可能变大。
- **字段防御**: `_load()` 时自动执行 `_ensure_fields()`，补全可能缺失的字段。
- **非副本返回**: `get_chat_history` 和 `get_project_history` 返回的是原始列表的切片引用，不是副本（与短期记忆的 `get_recent_context` 不同）。
- **delete_all 不清内存**: `delete_all()` 只删文件，不清 `self.data`。后续 `_save()` 会重建文件。
