# 记忆管理器 (MemoryManager)

**源文件**: `context/memory_manager.py`

## 概述

MemoryManager 是记忆系统的统一入口，组合了短期记忆（`ShortTermMemory`）和长期记忆（`LongTermMemory`）两个组件，对外提供统一的读写 API。它还负责调用 LLM 对长期聊天历史做摘要总结。

系统中只有 `cli.py` 直接实例化和使用 MemoryManager，其他组件（IntentionAgent、OrchestrationAgent）通过构造函数接收它的引用。

## 与两层记忆的关系

```
MemoryManager
├── self.short_term: ShortTermMemory  ← 纯内存，滑动窗口，会话级
└── self.long_term:  LongTermMemory   ← JSON 文件持久化，跨会话
```

写入时同时写两层，读取时按需从不同层取数据：

| 操作 | 短期记忆 | 长期记忆 |
|---|---|---|
| `add_message()` | 写入（带 metadata） | 写入（带 session_id） |
| `get_full_context()` | 读最近 5 轮对话 + 统计 | 读偏好 + 历史 + 项目记录 + 常用操作 |
| `get_context_for_agent()` | 读最近 3 轮对话字符串 | 读偏好 |
| `end_session()` | 清空 | 不动（数据已持久化） |

## 类: `MemoryManager`

### `__init__(self, user_id: str, session_id: str, storage_path: str = "data/memory", llm_model=None)`

初始化记忆管理器，创建两层记忆实例。

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `user_id` | `str` | (必填) | 用户 ID |
| `session_id` | `str` | (必填) | 当前会话 ID |
| `storage_path` | `str` | `"data/memory"` | 长期记忆的存储目录 |
| `llm_model` | 可调用对象 | `None` | LLM 模型实例，用于 `get_long_term_summary_async()`。传 `None` 则摘要功能不可用 |

**内部状态**:
- `self.user_id`, `self.session_id`, `self.llm_model`
- `self.short_term`: `ShortTermMemory(max_turns=10)` — 硬编码 10 轮
- `self.long_term`: `LongTermMemory(user_id, storage_path)`

**实际调用**（`cli.py:108`）:
```python
self.memory_manager = MemoryManager(
    user_id=self.user_id,
    session_id=self.session_id,
    llm_model=self.model
)
```

### `add_message(self, role: str, content: str, metadata: Dict = None)`

同时向短期记忆和长期记忆写入一条消息。这是系统中唯一的消息写入入口。

**输入**:

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `role` | `str` | (必填) | `"user"` 或 `"assistant"` |
| `content` | `str` | (必填) | 消息文本 |
| `metadata` | `Dict` | `None` | 传给短期记忆的额外元数据 |

**输出**: 无返回值。

**逻辑**:
1. 调用 `self.short_term.add_message(role, content, metadata)` — 带 metadata，不带 session_id。
2. 调用 `self.long_term.add_chat_message(role, content, self.session_id)` — 带 session_id，不带 metadata。

**调用时机**（`cli.py`）:
- 用户输入经过意图识别后：`add_message("user", user_input)` — 存的是用户原话
- 编排执行完成后：`add_message("assistant", json.dumps(result_data))` — 存的是最终结果 JSON

中间的 agent 调用和工具执行过程不存入记忆。

---

### `get_full_context(self) -> Dict[str, Any]`

获取两层记忆的完整上下文快照，用于状态展示。

**输入**: 无参数。

**输出**: `Dict[str, Any]`，结构如下：

```python
{
    "short_term": {
        "recent_dialogue": List[Dict],  # 最近 5 轮消息列表
        "context_string": str,          # 最近 5 轮的格式化字符串
        "statistics": Dict              # 短期记忆统计
    },
    "long_term": {
        "preferences": Dict,            # 所有偏好 {type: value}
        "chat_history": List[Dict],     # 最近 10 条聊天记录
        "project_history": List[Dict],  # 最近 5 条项目决策
        "frequent_actions": List[tuple],# 前 3 常用操作
        "statistics": Dict              # 长期记忆统计
    }
}
```

**逻辑**: 纯粹的数据聚合，分别调用两层记忆的查询方法，组装成嵌套字典返回。

**调用场景**: `cli.py` 的 `show_status()` 命令用它展示当前记忆状态。

---

### `get_context_for_agent(self, long_term_summary: str = None) -> str`

拼接用于 Agent prompt 的上下文字符串。

**输入**:

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `long_term_summary` | `str` | `None` | 预先生成的长期记忆摘要文本 |

**输出**: `str` — 格式化的上下文字符串，无内容时返回 `"无上下文信息"`。

**逻辑（按顺序拼接三个部分）**:

1. **历史会话总结**（若 `long_term_summary` 非空）:
   ```
   【历史会话总结】
   {long_term_summary}
   ```

2. **项目偏好**（若存在非空偏好）:
   ```
   【项目偏好】
   - target_market: B2B SaaS
   - tech_stack: Python + React
   ```

3. **当前会话对话**（若短期记忆非空，取最近 3 轮）:
   ```
   【当前会话对话】
   用户: 调研AI市场
   助手: 调研结果...
   ```

用 `\n` 连接所有部分。若三部分都为空则返回 `"无上下文信息"`。

---

### `end_session(self)`

结束当前会话。

**输入**: 无参数。

**输出**: 无返回值。

**逻辑**: 调用 `self.short_term.clear()` 清空短期记忆。长期记忆不做处理（数据已在每次写入时持久化）。

**调用时机**: 用户在 CLI 中输入 `exit` 时触发。

---

### `get_long_term_summary_async(self, max_messages: int = 50) -> str`

使用 LLM 对长期聊天历史进行摘要总结（异步方法）。

**输入**:

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `max_messages` | `int` | `50` | 最多纳入总结的消息数量 |

**输出**: `str` — LLM 生成的摘要文本（不超过 200 字）。无 LLM 或无历史时返回空字符串。

**逻辑**:
1. **前置检查**: 若 `self.llm_model` 为 `None`，直接返回 `""`。
2. **过滤当前会话**: 从长期记忆获取全部聊天记录，排除 `session_id` 等于当前会话的消息（只总结历史会话）。
3. **获取项目决策**: 从长期记忆获取最近 20 条项目决策记录。
4. **空数据检查**: 若过滤后的聊天记录和项目记录都为空，返回 `""`。
5. **构建 prompt**: 将聊天记录格式化为 `[timestamp] role: content`，项目记录格式化为 `[timestamp] action_type: description`，拼入总结 prompt。
6. **调用 LLM**: 以 `[{"role": "user", "content": prompt}]` 格式调用 `self.llm_model`。
7. **解析响应**: 处理三种返回格式：
   - 异步生成器（`__aiter__`）：迭代取最终 chunk
   - 含 `.content` 属性的对象：直接取 `str(response.content)`
   - 其他：`str(response)`
8. **异常处理**: 调用失败时记录错误日志，返回 `""`。

---

### `get_long_term_summary(self, max_messages: int = 50) -> str`

`get_long_term_summary_async` 的同步包装。

**输入**: 同 `get_long_term_summary_async`。

**输出**: 同 `get_long_term_summary_async`。

**逻辑**:
1. 尝试 `asyncio.get_running_loop()`。
2. 若已在 async 上下文中，记录 warning 日志并返回 `""`（避免嵌套事件循环）。
3. 若不在 async 上下文中，用 `asyncio.run()` 同步执行异步版本。

## 使用方式

### 基本用法

```python
from context.memory_manager import MemoryManager

mm = MemoryManager(user_id="user_001", session_id="session_abc", llm_model=model)

# 写入消息（同时写入两层记忆）
mm.add_message("user", "帮我调研竞品")
mm.add_message("assistant", "调研结果...")

# 获取 Agent 上下文
context_str = mm.get_context_for_agent(long_term_summary="用户关注AI教育方向")

# 获取完整状态
full = mm.get_full_context()

# 生成长期记忆摘要
summary = await mm.get_long_term_summary_async()

# 结束会话
mm.end_session()
```

### CLI 中的实际调用链

```
用户输入
  → cli._get_long_term_summary()
      → mm.long_term.get_preference()          # 读偏好
      → mm.get_long_term_summary_async()        # LLM 总结历史
      → mm.long_term.get_project_history()      # 读项目决策
  → cli 构建 context_messages 传给 IntentionAgent
  → IntentionAgent 返回意图
  → mm.add_message("user", user_input)          # 写入记忆
  → OrchestrationAgent 调度执行
  → mm.add_message("assistant", result_json)    # 写入记忆
```

### 直接访问内部组件

CLI 中多处直接访问 `mm.short_term` 和 `mm.long_term`，绕过 MemoryManager 的方法：

```python
mm.short_term.get_recent_context(n_turns=5)  # cli.py:172 — 获取上下文消息
mm.short_term.clear()                         # cli.py:674 — clear 命令
mm.long_term.get_preference()                 # cli.py:299 — 读偏好
mm.long_term.get_project_history(limit=None)  # cli.py:320 — 读项目历史
```

## 设计要点

- **双写策略**: `add_message()` 同时写入两层记忆，短期记忆带 `metadata`，长期记忆带 `session_id`，字段不完全一致。
- **会话隔离**: `get_long_term_summary_async()` 总结时自动排除当前 `session_id` 的消息，只总结历史会话。
- **LLM 可选**: `llm_model` 为 `None` 时摘要功能静默失效，不影响其他功能。
- **封装不完全**: CLI 中多处直接访问 `mm.short_term` 和 `mm.long_term`，MemoryManager 更像是一个组合容器而非严格的封装层。
